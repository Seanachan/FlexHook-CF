# Literature Review: LSTM/RNN for Temporal Modeling in Referring Multi-Object Tracking (RMOT)

## Research question

Has LSTM/RNN been used for temporal modeling in Referring Multi-Object Tracking (RMOT)? Specifically: (1) RMOT methods (Refer-KITTI / Refer-KITTI-V2 / Refer-Dance benchmarks; e.g. TransRMOT, TempRMOT, iKUN, MLS-Track, DeepRMOT, COAL, FlexHook) — do any use LSTM/RNN/GRU for temporal aggregation of trajectory or feature sequences, vs transformer temporal attention? (2) The broader trajectory of temporal modeling in multi-object tracking: classic LSTM/RNN-based MOT (RNN-MOT Milan 2017, AMIR Sadeghian 2017, Social-LSTM) and why the field shifted to transformers. (3) Any recent 2023-2026 work combining recurrent temporal modules with language-conditioned / referring tracking or referring video object segmentation. Prioritize real papers with verifiable citations (venue, year, arXiv id). Note clearly where evidence is absent.

## Executive summary

No published RMOT method on the Refer-KITTI / Refer-KITTI-V2 / Refer-Dance benchmarks uses an LSTM/RNN/GRU recurrent module for temporal aggregation of trajectory or feature sequences; they universally rely on transformer attention or pooling/Kalman mechanisms. Specifically, TransRMOT, TempRMOT, and TenRMOT (RMOT/RMOTS) use transformer-based temporal attention (query memories, temporal enhancement / query-update modules); iKUN uses temporal average pooling plus a learned-MLP Kalman filter, and tried LSTM/GRU **only for the Kalman-filter noise sub-networks** (not for temporal feature aggregation), finding "don't show obvious improvements" there; FlexHook uses a non-recurrent channel-concat + MLP "Temporal Integration" module and a masked cross-attention matcher (PCD) — with zero LSTM/RNN/GRU in the codebase. By contrast, the pre-transformer classic-MOT era (2017-2018) was foundationally recurrent: Milan et al. (AAAI 2017, RNN-MOT) was the first end-to-end RNN approach to online MOT (RNN for state estimation + LSTM for data association), Sadeghian et al. (AMIR, ICCV 2017) used RNN/LSTM for long-term multi-cue dependencies, and Kim et al. (ECCV 2018, Bilinear LSTM) used a recurrent appearance model. The field then shifted to transformers (TrackFormer CVPR 2022, MOTR ECCV 2022) explicitly because post-processing association and recurrent state propagation limited end-to-end temporal exploitation. Critically, evidence is ABSENT for any 2023-2026 RMOT or referring-video method that adopts a true recurrent (LSTM/RNN/GRU) temporal module — the only "recurrence-adjacent" mechanisms found (TenRMOT's momentum EMA query update, TempRMOT's FIFO query memory) are stateful but read via attention, not gated recurrent units.

### 1. - **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** Five primary sources, all 3-0 unanimous. iKUN: 'temporal model is realized by temporal average pooling' (T=8, stride 4) + spatial GAP; NKF = two fully-connected nets, not RNN. TempRMOT abstract: 'query-driven Temporal Enhancement Module that represents each object as a Transformer query, enabling long-term spatial-temporal interactions'; TEM = transformer self/cross-attention over a FIFO query memory; words LSTM/RNN/GRU appear nowhere. TenRMOT (arXiv:2410.13437): Query Update Module uses momentum EMA q^c_t=alpha*e_{t-1}+(1-alpha)*q^c_{t-1} + attention fusion at encode/decode; zero LSTM/RNN/GRU mentions. ### 2. iKUN (CVPR 2024) tried LSTM/GRU for its Neural Kalman Filter noise sub-networks (a motion/bbox-noise regression task) and kept plain fully-connected layers — NOT a test of recurrence for temporal feature aggregation.

- **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** Verbatim from iKUN Section 3.4 (Neural Kalman Filter): 'F_R and F_Q are designed as fully connected layers. More complex structures (e.g., LSTM and GRU) are also tried, but don't show obvious improvements.' R-Net/Q-Net dynamically adjust observation/process noise (R_k=F_R(z_k), Q_k=F_Q(x_{k-1})).

  **Scope / interpretation (do not overstate):** the LSTM/GRU trial was confined to the two small noise-regressor sub-networks F_R, F_Q — predicting Kalman observation/process-noise covariances — NOT the aggregation of a trajectory's appearance/language feature sequence (iKUN does that via temporal average pooling, T=8 stride 4, never tested with an LSTM). So this is only *tangential* evidence against "LSTM for temporal in RMOT": recurrence was found unhelpful for a tiny scalar-regression sub-problem where any sequence model is overkill. It is **not** evidence that LSTM fails for language-conditioned trajectory feature aggregation — that specific use remains untested by any RMOT work (see Finding 8). Earlier phrasing calling this iKUN "rejecting LSTM" overstated the result.

- **Sources:**

### 4. The classic pre-transformer MOT era (2017-2018) was foundationally RNN/LSTM-based for temporal modeling: Milan et al. (AAAI 2017, RNN-MOT) was the first end-to-end RNN approach to online multi-target tracking, using an RNN for state prediction/update/track-management and LSTMs for per-frame data association.

- **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** arXiv:1604.03635 abstract verbatim: 'We present a novel approach to online multi-target tracking based on recurrent neural networks (RNNs)... AAAI OJS keywords: 'Multi-target tracking, recurrent neural networks, long short-term memory, data association.' Architecture is recurrent-in-recurrent: outer RNN does continuous state estimation + track management per frame; inner LSTM solves combinatorial data association one target per step. Predates transformer MOT (TransTrack 2020). Multiple primary sources (arXiv, AAAI OJS, ACM DOI 10.5555/3298023.3298181), all 3-0.

### 5. Sadeghian et al. (AMIR, ICCV 2017) used a structure of RNNs (LSTM-based) to jointly reason over multiple cues (appearance, motion, interaction) over a temporal window, encoding long-term temporal dependencies to correct data-association errors and recover from occlusion — a canonical RNN/LSTM-era MOT method.

- **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** Abstract verbatim: 'we present a structure of Recurrent Neural Networks (RNN) that jointly reasons on multiple cues over a temporal window' and 'encodes long-term temporal dependencies across multiple cues' and 'correct many data association errors and recover observations from an occluded state.' Title: 'Tracking the Untrackable: Learning to Track Multiple Cues with Long-Term Dependencies.' This is generic MOT (not RMOT/referring), included as in-scope background per the research question. Two primary sources (arXiv + CVF), 3-0.

### 6. Kim, Li & Rehg (ECCV 2018) proposed the Bilinear LSTM, a recurrent network that temporally aggregates a track's appearance and motion history to score object tracks within a multiple-hypothesis-tracking framework — further evidence of LSTM/RNN temporal modeling in classic MOT.

- **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** Abstract verbatim: 'we propose a novel recurrent network model, the Bilinear LSTM, in order to improve the learning of long-term appearance models via a recurrent network' and 'We train an LSTM that can score object tracks based on both appearance and motion and utilize it in a multiple hypothesis tracking framework.' Title: 'Multi-object Tracking with Neural Gating Using Bilinear LSTM' (Springer LNCS doi:10.1007/978-3-030-01237-3_13). Primary CVF source, 3-0. Caveat: this is track-scoring/appearance temporal modeling, not motion-trajectory prediction per se, but it is recurrent aggregation over a track's frame sequence.

### 7. The MOT field shifted from recurrent (RNN/LSTM) to transformer temporal modeling, exemplified by TrackFormer (CVPR 2022) and MOTR (ECCV 2022), motivated by the limitation that heuristic post-processing association and recurrent state propagation prevented end-to-end exploitation of temporal variation.

- **Confidence:** high  |  **Adversarial vote:** 3-0

- **Evidence:** MOTR abstract verbatim: 'The post-processing nature of association prevents end-to-end exploitation of temporal variations in video sequence'; MOTR introduces a 'track query' extending DETR, 'transferred and updated frame-by-frame to perform iterative prediction over time'; its Temporal Aggregation Network is a modified transformer decoder layer (attention/MLP), not recurrent. TrackFormer abstract verbatim: 'Our model achieves data association between frames via attention by evolving a set of track predictions through a video sequence' with 'identity preserving track queries' propagated autoregressively via self/encoder-decoder attention, 'omitting any additional graph optimization or modeling of motion and/or appearance.' Both are general MOT (not RMOT) but establish the attention-replaces-recurrence trajectory. Two primary sources, 3-0.

- **Sources:**

  - https://arxiv.org/abs/2101.02702

  - https://arxiv.org/abs/2105.03247


### 8. No evidence was found of any 2023-2026 RMOT or referring-video method adopting a true gated recurrent (LSTM/RNN/GRU) temporal module; the only recurrence-adjacent mechanisms found are stateful-but-attention-read (TempRMOT's FIFO query memory, TenRMOT's momentum EMA query update), which are not RNN-family modules.

- **Confidence:** medium  |  **Adversarial vote:** 3-0

- **Evidence:** This is an absence-of-evidence finding. Across the RMOT methods examined (iKUN, TempRMOT, TenRMOT, FlexHook), none use LSTM/RNN/GRU; iKUN explicitly tested and rejected them. TempRMOT keeps a FIFO memory of past-frame transformer queries (stateful but read via attention, not recurrent state transition); TenRMOT's QUM uses a momentum exponential-moving-average update (q^c_t=alpha*e_{t-1}+(1-alpha)*q^c_{t-1}), recurrence-adjacent but not a gated RNN. Confidence is medium rather than high because the verified claim set did not include exhaustive coverage of every 2023-2026 RMOT/RVOS method (notably TransRMOT, MLS-Track, DeepRMOT, COAL were named in the question but lack their own verified primary-source claims here), so 'no method uses recurrence' is supported for the examined subset only.
